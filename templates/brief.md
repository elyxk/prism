# Launch Brief — [feature name]

Optional entry point for PRISM. Drop this file at `prism/inputs/brief.md` and run `Run Prism — Init from brief`. PRISM parses it for feature name, competitor URLs, and subreddits; asks you to fill in anything missing.

Write in prose — no schema. The paragraphs below are prompts, not required headings. Delete, reorder, or rewrite freely.

## What we're launching
<One or two sentences naming the feature and what it does. Kebab-case slug somewhere in here helps: e.g. "meeting-scheduler".>

## Who we're launching against
<Drop 3–5 competitor homepage URLs. A line each, or a comma-separated list, whatever's natural. If you only have names, that works too — PRISM will ask you for their hero headlines.>

Examples:
- https://competitor-a.com
- https://competitor-b.com
- https://competitor-c.com

## Where users talk about this problem
<Subreddits, Twitter hashtags, communities. "r/productivity, r/artificial" is enough. Skip if nothing comes to mind.>

## Anything else worth knowing
<Free-form. Past campaign angles that flopped, constraints from legal, a specific quote from a customer call — whatever context shaped this launch. PRISM reads this file in Phase 0.1 alongside everything else in inputs/, so nothing you write here is wasted.>
