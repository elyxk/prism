# PRISM — AI Messaging Agent

A Claude-powered agent that takes a product feature and any available data, and produces a complete, evidence-based messaging system — positioning, copy variants, a scored recommendation, and publication-ready landing pages.

**One command. Autonomous waterfall. One human checkpoint.**

---

## What It Produces

| Output | What it is |
|---|---|
| Signal Report | What your internal docs say vs what users actually say vs what competitors claim |
| User Truth Document | Evidence-based audience profile — behavioral + linguistic, no invented personas |
| Positioning Map | The gap nobody in your category is claiming, and your position claim |
| Messaging Framework | Core value prop, 3 pillars, narrative arc, voice constraints |
| 5 Copy Variants | Functional / Emotional / Pain-First / Outcome-First / Social Proof |
| Variant Review | All 5 variants scored with evidence, recommendation, override prompt |
| 2 HTML Pages | Winner + Runner Up — publication-ready, fully styled |

---

## How It Works

PRISM runs as a 7-phase waterfall. Each phase saves its output before the next reads it. The only human pause is before generating HTML — you review the scored variants and confirm or override.

```
Phase 0 — Signal Collection     (internal docs + competitors + community)
Phase 1 — Audience Definition   (behavioral + linguistic profiles)
Phase 2 — Positioning           (gap analysis + position claim)
Phase 3 — Messaging Architecture (value prop + pillars + narrative arc)
Phase 4 — Copy Variants         (5 angles, all evidence-tagged)
Phase 5 — Scoring               (comparative scoring, winner + test plan)
Phase 6 — Landing Pages         (review doc → 2 HTML files)
                ↑
         Only pause here
```

The same feature can be re-run with a new signal (Delta Run) to produce a different angle — without redoing the full pipeline.

## Install

PRISM is a Claude skill. Drop the `prism/` skill folder into your project's `.agents/skills/` directory.

**Via git (recommended):**
```bash
cd your-project
mkdir -p .agents/skills
git clone https://github.com/elyxk/prism.git .agents/skills/prism
```

**Manual:**
Download the repo and copy the folder so the layout looks like this:
```
your-project/
└── .agents/
    └── skills/
        └── prism/
            ├── SKILL.md
            ├── references/
            │   └── prism-phases.md
            └── templates/
                ├── config.txt
                ├── product-context.md
                ├── brand-voice.md
                └── brief.md
```

That's all the installation. Everything else is generated by the skill on first run.

## Quick Start

### 1. Kick off Init

Open Claude Code in your project root and say:

```
Run Prism — Init
```

Claude will ask you 4 quick questions:
1. What feature are we positioning?
2. Paste 3–5 competitor homepage URLs (Claude fetches the hero headlines for you).
3. Any subreddits to watch for user language?
4. Any Twitter/X keywords?

Claude then scaffolds `prism/` at your project root, writes `prism/config.txt` from your answers, and copies two templates into `prism/inputs/` for you to fill in:

- `prism/inputs/product-context.md` — 5 sentences about the product
- `prism/inputs/brand-voice.md` — tone, rules, banned words

**Already have a launch brief in Notion or Linear?** Paste it into `prism/inputs/brief.md` and run `Run Prism — Init from brief` instead. Claude parses it and only asks for what's missing.

### 2. Fill the two input files

Both templates have inline prompts. Should take 5 minutes each. You can do this in any editor.

### 3. Run the full pipeline

```
Feature: [your-feature-slug]
Dataset: [what's new — Reddit angle / analytics cohort / competitor observation]
Run Prism — Full
```

PRISM runs Phases 0–6 autonomously. It pauses exactly once — at Phase 6.3 — so you can review the 5 scored variants before it generates HTML.

## Run Modes

### Init — first-time setup (conversational)
```
Run Prism — Init
```
4-question interview → Claude writes `config.txt` and scaffolds the workspace.

### Init from brief — if you already have a one-pager
```
Run Prism — Init from brief
```
Reads `prism/inputs/brief.md`, extracts what it can, asks for the rest.

### Full Run — new feature or first run on a feature
```
Feature: [feature-slug]
Dataset: [what's new]
Run Prism — Full
```

### Delta Run — Audience Signal
New audience data on an existing feature (Reddit thread, analytics cohort, user interview, support tickets). Rebuilds User Truth; cascades through Phase 2 → 6. Outputs versioned (`signal-report-v2.md`), never overwritten.
```
Feature: [feature-slug]
Mode: Delta / Audience
New input: prism/inputs/[new-file]
Skip to: Phase 1
```

### Delta Run — Competitive Signal
New competitor, competitor repositioning, or a campaign that shifts the category. User Truth stays intact. Updates Competitor Intelligence Table and Phase 2, then decides whether to regenerate Phase 3 → 6 or just re-score.
```
Feature: [feature-slug]
Mode: Delta / Competitive
New input: [competitor URL or name + hero headline]
Skip to: Phase 2
```

## The MVP vs The Full Agent

PRISM is designed to run with minimum inputs and improve as you feed it more.

### Minimum to run (MVP)
- `product-context.md` — 5 sentences about the product
- `brand-voice.md` — tone, rules, banned words
- 3–5 competitor homepage URLs (Init fetches the hero headlines for you)

Output: valid positioning, 5 copy variants, 2 HTML pages.

### What makes it significantly better
| Addition | What it unlocks |
|---|---|
| Reddit thread or subreddits in config | Verbatim user language — frustration/relief vocabulary for headlines |
| Analytics data (CSV or Mixpanel API) | Behavioral audience profile — who activates, who drops off, and exactly where |
| Competitor Intelligence Skill output | Full position map — tone pattern, ignored audience, two-axis mapping |
| Brand Voice Skill output | Voice constraints applied to every word of copy |
| Style Skill output | HTML pages that match your actual brand design system |

### Connecting Mixpanel directly (advanced)
Instead of a CSV, connect PRISM to the Mixpanel API to read cohort data live:

```
Tool: Mixpanel API
Auth: API secret from Mixpanel project settings
Reads: /data/cohorts endpoint filtered by [feature_name]
Output: auto-saved to prism/inputs/analytics-export.csv for Phase 0
```
Same pattern works for Amplitude. When connected, Phase 1 (Audience Definition) is built on real behavioral data — activation rates, drop-off steps, power user profiles — rather than inference.

## The Standalone Skills

Three optional skills run once and save their output for all future PRISM runs.

| Skill | Output file | How often |
|---|---|---|
| Competitor Intelligence Skill | `competitor-intelligence-table.md` | Monthly |
| Brand Voice Skill | `brand-voice-document.md` | Once, update on rebrand |
| Style Skill | `company-style.css-variables` | Once, update on design refresh |

If any file is missing — PRISM flags it inline and continues. It never stops for a missing optional input.

---

## Running On a New Feature

PRISM is feature-agnostic. For a new feature:

1. Run `Run Prism — Init` again — the interview writes a fresh `config.txt` for the new feature.
2. Update `prism/inputs/product-context.md` with the new product description. If the new feature is in a different category, update `brand-voice.md` too; otherwise reuse.
3. Run: `Feature: [new-slug] / Dataset: [what's new] / Run Prism — Full`
4. Outputs save to `prism/outputs/[feature-name]/` — previous feature outputs untouched.

Brand voice and style files are shared across features. Competitor intelligence is per-category — re-run the Competitor Intelligence Skill if the new feature competes in a different market.

## The Human Checkpoint

After Phase 6.3, PRISM pauses and outputs:

> "Review `prism/outputs/[feature-name]/variant-review.md`. Confirm the recommended variants or specify your preference. I'll generate HTML when you're ready."

`variant-review.md` contains:
- All 5 variants with full copy
- Score breakdown with evidence for each criterion
- Why each angle could win and why it could fail
- The recommendation with test plan
- An explicit override prompt — you can reject the recommendation and choose different variants

This is the only moment in the pipeline where human judgment is required.

## Design Principles

- **No invented data.** Every claim traces to a source. Every copy element is tagged.
- **User language, verbatim.** Real words from real users convert better than paraphrases.
- **Flag and continue.** Missing inputs are noted inline. The pipeline never stops for optional inputs.
- **One checkpoint.** Everything before the HTML is autonomous. Human judgment enters once — after the evidence is visible.
- **Versioned, not overwritten.** Delta runs layer on top of previous runs. Nothing is lost.
